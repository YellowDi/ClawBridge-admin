# 用户表格操作聚合 Modal 设计

## 背景

用户列表当前在每行直接展示「可用知识库」「授权」「余额」「编辑」「删除」五个入口，操作列较宽且主次不清。目标是把前四项合并进一个通过分段切换的 Modal，并把删除收进纯图标「更多」入口。

## 方案

采用一个「用户设置」Modal 作为主入口。Modal 顶部显示当前用户名称，并通过 Tabs/segment 切换四个面板：

- 编辑：沿用现有用户详情加载与更新逻辑。
- 可用知识库：沿用现有知识库列表加载、勾选和保存逻辑。
- 授权：沿用现有模型授权和 Agent 授权逻辑。
- 余额：沿用现有余额查询、流水展示和余额调整逻辑。

用户表格操作列保留「设置」按钮和一个纯图标「更多」按钮。更多菜单只放危险操作「删除」，点击后打开现有删除确认 Modal。

## 范围

- 不新增后端 API。
- 不改用户、授权、余额、知识库的数据契约。
- 不新增运行时依赖。
- 不重做表格、筛选或用户创建流程。
