# 插件详情 Modal 极简摘要设计

## 背景

插件详情当前把基础字段、能力清单与两段 JSON 同时展开，信息密度不均，运营人员需要反复浏览低频排障信息。目标是突出插件是否可用、版本、来源和能力，其他内容按需查看。

## 方案

使用 HeroUI `Modal` 展示详情。顶部展示名称、稳定 ID、版本、状态和来源，描述紧随其后作为摘要。

正文只保留非空的能力分类：工具、Channel、Provider、Hook、Command。能力继续使用现有标签组件，不增加装饰性卡片。

文件大小、更新时间、SHA-256、来源地址与创建时间收进默认折叠的“更多信息”。Manifest 与 Package 合并进默认折叠的“原始配置”，展开后继续展示格式化 JSON。

底部操作保持现有行为：安装为主操作，编辑和删除为次级操作；已删除版本不展示这些操作。

## 范围

- 只调整 `PluginDetailModal` 与其详情展示辅助组件。
- 不修改插件数据接口、详情加载逻辑或安装、编辑、删除行为。
- 不新增运行时依赖，不改变页面其他布局。

## 验证

- 运行 `pnpm exec tsc --noEmit`。
- 运行 `pnpm exec eslint .`。
- 使用已登录本地应用检查抽屉默认信息、两处展开区及底部操作。
