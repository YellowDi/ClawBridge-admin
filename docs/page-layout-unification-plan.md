# 普通管理页面布局统一计划

## 目标

统一概览与模型之外的普通管理页面，让同类页面共享稳定的操作层级、内容工具栏、集合布局和状态反馈，同时保持现有 API、业务流程、表单字段、权限逻辑和弹窗行为不变。

领域术语与页面形态以根目录 `CONTEXT.md` 为准。

## 页面归类

| 页面形态               | 路由                                                                 |
| ---------------------- | -------------------------------------------------------------------- |
| 卡片集合页             | `/agents`、`/skills`、`/knowledge-bases`、`/subscriptions`           |
| 数据记录页             | `/users`、`/usage`、`/conversations`、`/plugins`、`/audit`、`/tools` |
| 配置详情页             | `/agents/[id]`、`/mcp`、`/settings`、`/security`                     |
| 特殊页面，不在本次范围 | `/`、`/models`                                                       |

## 统一规则

### 页面层级

- 顶部导航栏标题是普通页面唯一的一级标题。
- 页面级刷新、新建、上传、导入、保存、下发和导出进入页面操作区。
- 页面级 Tabs 位于顶部导航中间区域；搜索、筛选、排序和集合视图切换位于内容工具栏。
- 内容区按“可选统计摘要、内容工具栏、集合内容”的顺序组织。
- 单一集合不重复显示页面标题、视图标题或表格小标题；移除用量表格上方的重复标题。

### 页面操作区

- 次级操作在左，主操作在最右，保持单行。
- 窄屏下刷新优先改为纯图标按钮；主操作优先保留文字。
- 页面级操作超过三个时，只保留一到两个主要操作，其余进入纯图标更多菜单。
- 局部刷新和条目操作留在对应分区或条目中。

### 内容工具栏

- 桌面端筛选在左、搜索在右；移动端纵向排列。
- 搜索统一使用 `SearchField`，保留搜索图标和清除能力。
- 数据来源可以决定即时过滤或提交查询，但视觉与键盘交互保持一致。
- 不为没有搜索或筛选能力的页面增加空工具栏，也不在本次新增功能。
- 知识库与订阅套餐卡片化后增加可见排序选择器，以保留原表格排序能力。

### 卡片集合页

- 统一使用一列、两列、三列的响应式卡片集合，不使用四列。
- 集合卡片顶部显示名称、标识和主要状态，右上角显示卡片操作。
- 中部显示说明、标签或摘要；底部显示版本、大小、来源、更新时间、价格、额度等元数据。
- 常用操作直接显示，删除等危险操作进入纯图标更多菜单。
- 有详情页时整卡进入详情；卡片操作独立响应，不触发详情跳转。
- 同页卡片等高，不强制跨页面卡片等高。
- 统计摘要按管理价值可选，不为视觉一致性补装饰性指标。

### 数据记录页

- 统一由可选统计摘要、内容工具栏、DataGrid 和分页信息组成。
- 表格直接作为主要内容，不添加装饰性 Card 包裹。
- 常用行操作直接显示，危险行操作进入纯图标更多菜单。
- 页面视图 Tabs 与表格筛选 Tabs 分别归入顶部导航和内容工具栏。

### 配置详情页

- 保持统一的 `max-w-7xl` 页面宽度。
- 具有独立语义和操作边界的内容使用配置分区；普通说明、列表和表格不额外套 Card。
- 全页操作进入页面操作区，单个分区的保存或刷新进入分区右上角。
- 配置分区按内容使用单列或双列，不强制统一列数。

### 状态与间距

- 页面主要区域使用 `gap-6`；工具栏、卡片集合和卡片内部使用 `gap-4`；紧密相关的表格、错误和分页使用 `gap-3`。
- 卡片集合首次加载使用匹配最终列数和高度的骨架。
- 数据记录页继续使用 DataGrid 的加载与空状态能力。
- 空数据与筛选无结果使用不同文案。
- 首次加载失败显示内联错误和 Toast；刷新失败保留旧数据与用户输入。

## 逐页改造

### 卡片集合

- **Agent 编排**：刷新移入页面操作区；改用共享内容工具栏；状态移到名称旁；操作移到右上角；保留整卡详情入口和独立操作。
- **Skill 管理**：刷新移入页面操作区；移除“私有 Skill”重复标题；搜索与筛选改用共享内容工具栏和 `SearchField`；现有 Skill 卡片作为其他集合卡片的结构基准。
- **知识库**：DataGrid 改为卡片集合；保留统计摘要、状态、说明、来源、片段数、大小、更新时间和重试流程；增加可见排序选择器。
- **订阅套餐**：DataGrid 改为卡片集合；保留统计摘要、状态、说明、功能介绍、价格、席位、额度窗口摘要和编辑删除流程；增加可见排序选择器。

### 数据记录

- **用户、会话记录**：对齐共享内容工具栏、页面间距和操作归属。
- **用量统计**：视图 Tabs 移入顶部导航；移除表格上方“按日用量 / 按用户用量”重复小标题。
- **插件管理**：保留顶部视图 Tabs，统一两个视图的内容工具栏、间距和行操作模式。
- **审计日志、工具与权限**：当前保留真实接口缺失状态；外层结构按未来数据记录页对齐，不添加静态假数据。

### 配置详情

- **Agent 详情**：保留顶部 Tabs，统一页面操作和配置分区间距，不改变编辑、知识库和版本流程。
- **OpenClaw MCP**：保留配置详情定位，统一全页操作、局部操作和分区结构；表格不因位于分区内而增加额外装饰层。
- **系统设置、安全策略**：统一页面操作、配置分区和状态展示，不改变配置下发或接口缺失语义。

## 最小共享实现

- 继续使用 `AdminPage` 承载页面操作与页面级导航。
- 在 `components/admin-page-kit.tsx` 增加共享内容工具栏和卡片集合布局。
- 不新增万能卡片组件；各实体卡片保留自己的字段与操作。
- 不封装新的表格组件；继续直接使用 HeroUI Pro `DataGrid`。
- 不增加运行时依赖。

## 实施顺序

1. 增加共享内容工具栏与卡片集合，并收紧顶部页面操作区的响应式行为。
2. 统一 Agent 与 Skill，先验证共享卡片和工具栏基准。
3. 将知识库与订阅套餐转换为卡片，并保留排序与现有操作。
4. 统一数据记录页的工具栏、标题、Tabs、间距和危险操作。
5. 统一配置详情页的外层和配置分区。
6. 执行全路由桌面端与移动端视觉回归。

## 验证

- `pnpm exec tsc --noEmit`
- `pnpm exec eslint .`
- `pnpm build`
- `git diff --check`
- 浏览器逐页验证所有普通路由的顶部操作、工具栏、卡片或表格、加载、错误、空状态和响应式布局。
- 重点验证整卡详情入口与右上角操作不会互相触发，知识库和订阅套餐排序能力没有回归。

## 明确不做

- 不修改概览和模型页面。
- 不修改 API、请求参数、权限、表单字段和业务流程。
- 不新增知识库或订阅套餐详情页。
- 不为知识库或订阅套餐新增搜索筛选能力。
- 不添加新的 UI 框架、运行时依赖、万能页面配置或 schema 驱动布局。
